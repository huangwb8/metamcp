# tests

本目录用于存放 MetaMCP 项目的可复用测试代码、验证脚本、测试夹具与相关辅助资源。

约定：

- 这里放“改完之后还值得再次运行”的测试资产。
- AI 或开发者在修改项目后，应优先基于这里的代码判断项目是否仍可正常运行。
- 需要长期保留、可复用的验证逻辑放这里，不要散落到业务目录。
- 测试执行过程中产生的日志、快照、抓包、临时导出文件等中间产物，统一写入 `../tmp/`。

当前仓库级测试入口：

- `pnpm test`：运行根目录 `tests/` 下的仓库级契约测试
- `pnpm test:watch`：监听模式运行仓库级测试
- `pnpm test:coverage`：生成覆盖率报告到 `tmp/vitest/coverage/`

当前首批覆盖重点：

- MetaMCP 前后端共享的工具命名约定
- MetaMCP 函数式中间件组合行为
- `packages/zod-types` 中与 MCP server / namespace 刷新直接相关的共享协议
- 前端表单错误翻译逻辑
- `packages/trpc` 中 config router 的鉴权与输入边界

说明：

- 这些测试位于 `tests/contracts/`，目标是尽快发现跨包回归，而不是替代各子包内部更细粒度的单元测试。
- `apps/backend/src/**/*.test.ts` 这类包内测试仍然保留，用于贴近实现细节的验证；仓库级测试负责补“跨包契约”和“根目录可直接复跑”的那部分空白。
